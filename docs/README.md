# cumulo

A tool for reaching half-remembered resources fast, by association from just a few clues.

- Examples: cloud consoles, team portals. It cuts the daily cost of searching, recalling, and getting there.
- The premise: classifying information ahead of time is what makes association work. That act of organizing is valuable in itself (e.g. sharing a classification as onboarding material).
- So the core concern is not storing data but **classification**, and the **filtering (association)** that follows from it.

## About these docs

These docs hold only cumulo's **domain knowledge** — why the system is shaped the way it is, and the **vocabulary** worth sharing.
Anything the implementation or tests already guarantee (behavioral specs, validation rules, module layout) is left out.
